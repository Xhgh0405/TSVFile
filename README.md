# 上課練習(7) - TSV資料檔讀取程式

## 一、專案簡介

本專案是「上課練習(7) - TSV資料檔讀取程式」的完成版本。

程式使用 C# Windows Forms 製作，主要功能是讀取 `.tsv` 或 `.txt` 資料檔，並將檔案內容顯示在 `ListView` 表格中。

本程式主要練習：

- Windows Forms 介面設計
- MenuStrip 功能表
- OpenFileDialog 開啟檔案
- ListView 顯示資料
- StatusStrip 顯示狀態訊息
- 自訂類別與集合類別
- 文字檔讀取
- TSV 資料格式處理
- Ctrl + F 搜尋功能

本版本額外新增 `Ctrl + F` 搜尋功能，可以在目前載入的資料中快速搜尋文字，也可以使用 `F3` 搜尋下一筆符合的資料。

## 五、Search 功能表說明

`Search` 功能表是本版本新增的搜尋功能。

包含以下功能：

| 功能 | 快捷鍵 | 說明 |
|---|---|---|
| Find | Ctrl + F | 搜尋目前載入資料中的文字 |
| Find Next | F3 | 搜尋下一筆符合的資料 |

### Find

點選：

Search -> Find

或按下快捷鍵：

Ctrl + F

會開啟搜尋輸入視窗。

使用者輸入關鍵字後，程式會在目前已載入的資料中搜尋符合的文字。

搜尋範圍包含：

- 單字
- 音標
- 音檔路徑
- 解釋

只要其中一個欄位包含輸入的關鍵字，程式就會判定該筆資料符合搜尋條件，並自動選取該列。

### Find Next

點選：

Search -> Find Next

或按下快捷鍵：

F3

可以搜尋下一筆符合相同關鍵字的資料。

如果已經搜尋到最後一筆，程式會自動從第一筆資料重新開始搜尋。

## 八、資料格式說明

本程式主要讀取 TSV 格式資料。

TSV 是 `Tab Separated Values` 的縮寫，意思是欄位之間使用 Tab 分隔。

一行資料代表一筆資料。

每一欄之間要使用 Tab 分隔，不是一般空白。

範例：

abacus    ˋæbəkəs    Sound\A\abacus.mp3    一種計算工具
abandon   əˋbændən   Sound\A\abandon.mp3   放棄
ability   əˋbɪlətɪ   Sound\A\ability.mp3   能力

實際檔案中，欄位之間應該是 Tab。

如果資料不是用 Tab 分隔，程式仍然可以讀取檔案，但是資料可能無法正確分到不同欄位。

## 十、編碼支援說明

本程式支援 UTF-8 與有 BOM 的 Unicode 文字檔。

支援的編碼包含：

- UTF-8
- UTF-8 with BOM
- UTF-16 Little Endian
- UTF-16 Big Endian

因為 `WordCards.txt` 可能包含音標，例如：

- ˋæbəkəs
- əˋbændən
- əˋbɪlətɪ

如果文字編碼不正確，音標可能會顯示成亂碼。

因此本版本加入自動判斷 BOM 的讀檔方式，可以降低音標亂碼的問題。

## 十二、Ctrl + F 搜尋功能說明

本版本新增 `Ctrl + F` 搜尋功能。

使用者可以在目前載入的資料中搜尋指定文字。

搜尋時會檢查每一列資料的所有欄位。

搜尋範圍包含：

- 單字
- 音標
- 音檔路徑
- 解釋

只要某一欄包含關鍵字，就會被視為搜尋成功。

搜尋成功後，程式會自動選取符合的資料列，並將畫面捲動到該筆資料的位置。

如果找不到資料，狀態列會顯示找不到的提示訊息。

---

## 十三、F3 搜尋下一筆功能說明

使用 `Ctrl + F` 搜尋後，程式會記住最後一次輸入的關鍵字。

如果想繼續搜尋下一筆符合資料，可以按：

F3

例如搜尋 `a`，可能會有很多筆資料符合。

第一次搜尋會找到第一筆。

按下 `F3` 後，會繼續找下一筆。

如果搜尋到最後一筆後再按 `F3`，程式會回到第一筆繼續搜尋。

## 十六、為什麼沒有加入播放聲音功能

本版本沒有加入播放聲音功能。

原因是這個專案主要是「TSV / TXT 資料檔讀取程式」，不一定每一個 `.txt` 檔案都是英文單字資料，也不一定每筆資料都有音檔路徑。

如果加入播放聲音功能，當使用者開啟一般文字檔時，可能會造成使用上的混淆。

因此本版本只新增較通用的功能：

- Ctrl + F 搜尋
- F3 搜尋下一筆

這樣比較適合各種 `.txt` 或 `.tsv` 資料檔。

## 十八、常見問題

### Q1：為什麼開啟 txt 後資料沒有正確分欄？

可能是檔案內容不是使用 Tab 分隔。

本程式主要讀取 TSV 格式資料，因此欄位之間需要使用 Tab 分隔。

如果使用一般空白分隔，程式可能無法正確判斷欄位。

### Q2：為什麼音標顯示亂碼？

可能是文字檔編碼不正確。

建議將檔案另存成：

- UTF-8
- Unicode 文字

本版本已支援 UTF-8 與有 BOM 的 Unicode 文字檔，可以減少音標亂碼問題。

### Q3：為什麼 Ctrl + F 找不到資料？

可能原因如下：

- 尚未開啟檔案
- 輸入的關鍵字不存在
- 資料沒有成功載入
- 檔案格式不是 Tab 分隔格式

請先確認資料已經成功顯示在 `ListView` 中。

### Q4：為什麼沒有播放聲音功能？

因為本版本只保留通用的資料讀取與搜尋功能。

不是每個 `.txt` 檔案都一定是英文單字資料，也不一定有音檔路徑，所以沒有加入播放聲音功能。

### Q5：如果 Visual Studio 顯示 .NET Framework 4.8 未安裝怎麼辦？

請安裝：

.NET Framework 4.8 Developer Pack

或到專案屬性中，將 Target Framework 改成電腦已安裝的 .NET Framework 版本。

## 二十一、結論

本專案完成 TSV / TXT 資料檔讀取功能，並能將讀取到的資料顯示在 `ListView` 表格中。

除了完成基本的開檔與顯示功能外，本版本也新增 `Ctrl + F` 搜尋功能與 `F3` 搜尋下一筆功能，使使用者可以更方便地查找資料。
