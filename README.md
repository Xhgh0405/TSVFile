# 上課練習(7) - TSV資料檔讀取程式

## 一、專案簡介

本專案是「上課練習(7) - TSV資料檔讀取程式」的完成版本。

程式使用 C# Windows Forms 製作，主要功能是讀取 `.tsv` 或 `.txt` 資料檔，並將檔案內容顯示在 `ListView` 表格中。

本程式主要練習：

- Windows Forms 介面設計
- MenuStrip 功能表
- OpenFileDialog 開啟檔案
- ListView 顯示資料
- StatusStrip 顯示狀態訊息
- 自訂類別與集合類別
- 文字檔讀取
- TSV 資料格式處理
- Ctrl + F 搜尋功能

本版本額外新增 `Ctrl + F` 搜尋功能，可以在目前載入的資料中快速搜尋文字，也可以使用 `F3` 搜尋下一筆符合的資料。

## 四、功能表說明

### 1. File 功能表

`File` 功能表主要負責檔案開啟與程式結束。

包含以下功能：

| 功能 | 說明 |
|---|---|
| Open | 開啟 .txt 或 .tsv 資料檔 |
| Exit | 結束程式 |

### Open

點選：

File -> Open

可以選擇要開啟的資料檔。

支援的檔案格式包含：

- `.txt`
- `.tsv`

開啟檔案後，程式會讀取檔案內容，並將資料顯示在主畫面的 `ListView` 中。

本版本已將開啟檔案視窗的篩選條件修改為預設同時顯示 `.txt` 與 `.tsv`，因此可以直接看到 `WordCards.txt`，不需要手動切換檔案類型。

### Exit

點選：

File -> Exit

可以結束程式。

當使用者要關閉程式時，會跳出確認訊息：

確定要離開嗎？

如果選擇 `Yes`，程式會關閉。

如果選擇 `No`，程式會取消關閉。

## 五、Search 功能表說明

`Search` 功能表是本版本新增的搜尋功能。

包含以下功能：

| 功能 | 快捷鍵 | 說明 |
|---|---|---|
| Find | Ctrl + F | 搜尋目前載入資料中的文字 |
| Find Next | F3 | 搜尋下一筆符合的資料 |

### Find

點選：

Search -> Find

或按下快捷鍵：

Ctrl + F

會開啟搜尋輸入視窗。

使用者輸入關鍵字後，程式會在目前已載入的資料中搜尋符合的文字。

搜尋範圍包含：

- 單字
- 音標
- 音檔路徑
- 解釋

只要其中一個欄位包含輸入的關鍵字，程式就會判定該筆資料符合搜尋條件，並自動選取該列。

### Find Next

點選：

Search -> Find Next

或按下快捷鍵：

F3

可以搜尋下一筆符合相同關鍵字的資料。

如果已經搜尋到最後一筆，程式會自動從第一筆資料重新開始搜尋。

## 八、資料格式說明

本程式主要讀取 TSV 格式資料。

TSV 是 `Tab Separated Values` 的縮寫，意思是欄位之間使用 Tab 分隔。

一行資料代表一筆資料。

每一欄之間要使用 Tab 分隔，不是一般空白。

範例：

abacus    ˋæbəkəs    Sound\A\abacus.mp3    一種計算工具
abandon   əˋbændən   Sound\A\abandon.mp3   放棄
ability   əˋbɪlətɪ   Sound\A\ability.mp3   能力

實際檔案中，欄位之間應該是 Tab。

如果資料不是用 Tab 分隔，程式仍然可以讀取檔案，但是資料可能無法正確分到不同欄位。

## 九、WordCards.txt 說明

專案中已放入測試資料檔：

WordCards.txt

此檔案可用來測試程式是否能正確讀取資料、顯示音標、顯示解釋，以及使用搜尋功能。

`WordCards.txt` 已設定為建置時複製到輸出目錄。

Visual Studio 中的設定為：

Copy to Output Directory: Copy if newer

中文介面可能顯示為：

複製到輸出目錄：有更新時才複製

這樣執行程式時，`WordCards.txt` 會被複製到輸出資料夾，方便程式測試開啟檔案。

## 十、編碼支援說明

本程式支援 UTF-8 與有 BOM 的 Unicode 文字檔。

支援的編碼包含：

- UTF-8
- UTF-8 with BOM
- UTF-16 Little Endian
- UTF-16 Big Endian

因為 `WordCards.txt` 可能包含音標，例如：

- ˋæbəkəs
- əˋbændən
- əˋbɪlətɪ

如果文字編碼不正確，音標可能會顯示成亂碼。

因此本版本加入自動判斷 BOM 的讀檔方式，可以降低音標亂碼的問題。

## 十一、使用方式

### 步驟 1：開啟專案

使用 Visual Studio 開啟：

TSVFile.sln

### 步驟 2：執行程式

按下 Visual Studio 上方的執行按鈕，或按鍵盤：

F5

### 步驟 3：開啟資料檔

程式啟動後，點選：

File -> Open

接著選擇要開啟的檔案，例如：

WordCards.txt

也可以選擇其他 `.txt` 或 `.tsv` 檔案。

### 步驟 4：查看資料

開啟檔案後，資料會顯示在 `ListView` 表格中。

表格會顯示：

- 單字
- 音標
- 音檔路徑
- 解釋

### 步驟 5：搜尋資料

按下：

Ctrl + F

輸入要搜尋的關鍵字。

例如：

abacus

如果找到符合的資料，程式會自動選取該資料列。

### 步驟 6：搜尋下一筆

如果有多筆資料符合搜尋條件，可以按：

F3

程式會搜尋下一筆符合的資料。

## 十二、Ctrl + F 搜尋功能說明

本版本新增 `Ctrl + F` 搜尋功能。

使用者可以在目前載入的資料中搜尋指定文字。

搜尋時會檢查每一列資料的所有欄位。

搜尋範圍包含：

- 單字
- 音標
- 音檔路徑
- 解釋

只要某一欄包含關鍵字，就會被視為搜尋成功。

搜尋成功後，程式會自動選取符合的資料列，並將畫面捲動到該筆資料的位置。

如果找不到資料，狀態列會顯示找不到的提示訊息。

---

## 十三、F3 搜尋下一筆功能說明

使用 `Ctrl + F` 搜尋後，程式會記住最後一次輸入的關鍵字。

如果想繼續搜尋下一筆符合資料，可以按：

F3

例如搜尋 `a`，可能會有很多筆資料符合。

第一次搜尋會找到第一筆。

按下 `F3` 後，會繼續找下一筆。

如果搜尋到最後一筆後再按 `F3`，程式會回到第一筆繼續搜尋。
